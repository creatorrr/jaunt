### Task 2: Finding 28 interim — hard gate when specs span source roots or bypass the output root

**Files:** Modify `src/jaunt/cli.py` (`_discover_build_context` :2150-2165 area; apply the same gate on the check/status/clean/migrate discovery paths — the sibling `package_dir = next(...)` sites at :2684, :2701, :3195, :3504, :4793 all funnel through discovery helpers; put the gate in ONE shared helper called after discovery, not 6 copies). **Behavior:** After discovery, compute each governed spec module's OWNING root = the most-specific (longest-path) configured source root containing its `source_file`. Then:
1. If owning roots span >1 distinct root → `JauntConfigError` (exit 2): `"governed specs span multiple source_roots (<a>, <b>): jaunt 1.5 routes all generated output to the first existing root, which breaks packages under the others (FEEDBACK finding 28). **Tests:** default-roots project passes; two-root fixture with specs under both roots exits 2 with the finding-28 message; single-owning-root-but-wrong-order fixture exits 2 with the reorder message; check/status hit the same gate (not just build).

### Task 3: Finding 27 — sanctioned third-party imports + fingerprint drift re-stamps free

**Files:** Modify `src/jaunt/prompts/build_module.md:34`; modify `src/jaunt/builder.py` (`plan_refreeze_or_rebuild` :493-575). Test: `tests/test_fingerprint_restamp.py` (new) + prompt-text assertion in an existing prompts test if one exists.

**Part A — the rule.** Replace line 34's `- Only import dependencies listed above — do not guess or fabricate module paths.` with:
```
- Import spec-registry dependencies ONLY from the declared paths listed above — do not guess or fabricate module paths.
- The Python stdlib, and installed third-party distributions that the spec module itself imports or the owning package declares, are fair game: import them plainly from their real modules (no duck-typed stand-ins, no dynamic-import contortions).
- For anything else, inline the logic and mark it with JAUNT-NEEDS-DEP as instructed.
```
(Wording may be tuned; the three semantic points are fixed. This aligns the prompt with Task 1's validator.)

**Part B — free re-stamp on fingerprint-only drift.** Editing the template changes the generation fingerprint and restales every adopter module. Required behavior: a stale module whose spec snapshots are ALL unchanged (`classify_change(...) == "none"` for every entry) must resolve via `refreeze_module` — deterministically, with ZERO semantic-gate/model calls — stamping the new `generation_fingerprint` over the untouched body. In `plan_refreeze_or_rebuild`, short-circuit BEFORE the gate: if every entry classifies `"none"`, refreeze directly (do not add to `gate_modules`). Verify whether this already holds; if it does, the test still lands as the regression pin.

**Tests (failing first):** build a mocked-backend fixture; recompute staleness under a different `generation_fingerprint` (e.g. changed `[build] instructions`); assert the module lands in `refrozen` (not `rebuild`), the rewritten header carries the NEW fingerprint, the body is byte-identical, and the backend + gate were never invoked.
