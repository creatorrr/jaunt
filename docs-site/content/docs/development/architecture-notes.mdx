The repo keeps design and implementation notes under `docs/`:

- `docs/superpowers/specs/` — one design doc per feature (contract mode,
  whole-class magic, the Codex engine, repo-context subsystem, default builder
  skills, held-out tests, smart change detection, the background daemon,
  contract adoption parity, agent-friendly progress).
- `docs/superpowers/plans/` — the implementation plans that accompanied those
  designs.
- `docs/principles/2026-06-29-building-with-coding-agents.md` — the
  coding-agent principles framework: six core laws plus named corollaries that
  shape Jaunt's design (spec + verification carry the value, deterministic
  verdicts, untrusted model inputs, and so on).
- `docs/hooks.md` — the `jaunt guard` PreToolUse hook for coding agents.

These are useful when changing behavior or extending features; the docs in this site focus on the user-facing surface area.
