---
title: Configuration (jaunt.toml)
description: "Config keys and behavior."
---

Jaunt discovers the project root by walking upward from the current working directory until it finds `jaunt.toml`.

Minimal config:

```toml
version = 1

[paths]
source_roots = ["src"]
test_roots = ["tests"]
generated_dir = "__generated__"
```

Full config (all keys optional except `version`):

```toml
version = 1

[paths]
# Directories to scan for build specs (relative to project root).
source_roots = ["src", "."]
# Directories to scan for test specs (relative to project root).
test_roots = ["tests"]
# Directory name inserted into import paths and used on disk.
# Must be a valid Python identifier. Recommended: "__generated__".
generated_dir = "__generated__"

[llm]
# Informational under the Codex engine. The model that actually runs is set in
# [codex] below, and Codex authenticates via `codex login` / CODEX_API_KEY —
# Jaunt does not read an api_key_env for generation.
provider = "openai"
model = "gpt-5.5"
# Optional hard ceiling for accumulated API spend during a build/test command.
# max_cost_per_build = 5.0

[build]
# Max parallel workers for build generation.
jobs = 8
# Best-effort dependency inference (explicit deps always apply).
infer_deps = true
# Extra ty-based repair passes after the normal retry loop.
ty_retry_attempts = 1
# Pytest marker backend for generated async tests: "asyncio" or "anyio".
async_runner = "asyncio"
# Include targeted test-spec source in build prompts.
include_target_tests = false
# Validate that generated modules only import what their spec context allows.
check_generated_imports = true
# Extra import roots to allow in generated modules.
generated_import_allowlist = []
# Optional project-wide steering appended to build prompts.
# instructions = ["Prefer small composable helpers."]

[test]
# Max parallel workers for test generation.
jobs = 4
# Best-effort dependency inference for tests.
infer_deps = true
# Passed through to pytest as repeated "--pytest-args" flags.
pytest_args = ["-q"]
# Auto-generate pytest coverage for whole-class @jaunt.magic specs.
auto_class_tests = false

[daemon]
# Seconds between HEAD polls.
poll_interval = 2.0
# Max background jobs; 0 uses build.jobs.
max_jobs = 0
# Optional shell command run after each proposed, landed, parked, or failed job.
notify_command = ""
# Landing mode for green jobs. false (default): park as a reviewable proposal,
# landed explicitly with `jaunt jobs land`. true: auto-commit on green (pre-1.2.0
# behavior).
auto_commit = false

[agent]
# Code-generation engine. "codex" is the only supported value.
engine = "codex"

[codex]
# Model passed to `codex exec --model`.
model = "gpt-5.5"
# Reasoning effort: "low", "medium", or "high".
reasoning_effort = "high"
# Codex sandbox policy (`codex exec --sandbox`).
sandbox = "workspace-write"
# Include the installed codex CLI version in the freshness fingerprint.
fingerprint_cli_version = true
# Optional Codex feature flags.
# features = []
# Raw `-c key=value` passthrough to `codex exec` (advanced).
# [codex.config]

[skills]
# Auto-generate PyPI helper skills for imported libraries during build.
auto = true
# Seed Jaunt's bundled builtin skills into the Codex workspace.
builtin = true
# The default builtin set (override to trim/extend).
builtin_skills = [
  "asyncpg", "dbos", "descope", "fastmcp", "openai", "pydantic", "pydantic-ai",
  "pytest", "ruff", "spacy", "starlette", "ty", "uv",
]
# Retained for back-compat; unused by the Codex builder (skills are seeded,
# not injected as prompt text).
# max_chars_per_skill = 8000
# inject_user_skills = []

[context]
# Maintain treedocs.yaml and inject a repo map into build prompts.
repo_map = true
repo_map_file = "treedocs.yaml"
# Opt-in: LLM-enrich descriptions (else AST-only, offline).
enrich = false
# Cap the injected repo-map block.
max_chars = 6000
# Opt-in: model-written architecture overview injected into build prompts,
# digest-cached to .jaunt/PROJECT_OVERVIEW.md.
overview = false

[context.search]
# colgrep (LightOn next-plaid) semantic retrieval; requires `colgrep` on PATH.
enabled = false
# Jaunt queries `colgrep --json` and seeds _context/relevant_*.py.
internal_retrieval = true
max_hits = 8

[contract]
# Where derived contract batteries are written.
battery_dir = "tests/contract"
# Case kinds derived from docstring prose.
derive = ["examples", "errors"]
# Run mutation-based strength scoring at reconcile.
strength = true

[semantic_gate]
# Gate behaviorally-equivalent docstring edits before a full rebuild.
enabled = true
# Small model that judges contract equivalence (must run via `codex exec`).
model = "gpt-5.4-mini"
reasoning_effort = "high"   # low | medium | high

[prompts]
# If set, these are treated as file paths and read at runtime by the engine.
# Leave empty to use packaged defaults under src/jaunt/prompts/.
build_preamble = ""         # the always-on Jaunt preamble (codex_preamble.md)
build_system = ""
build_module = ""
test_system = ""
test_module = ""
project_overview_system = ""  # used when [context] overview = true
project_overview_user = ""
```

Notes:

- `paths.source_roots`: the CLI picks the *first existing* source root as the output base for generated build modules.
- `prompts.*` are treated as *file paths* and read at runtime by the engine.
- Jaunt is batteries-included: `rich`, `watchfiles`, `pytest`, `pytest-asyncio`, and `anyio` ship as core dependencies. There is nothing to install for watch mode, tests, or rich output — and no optional extras.
- Generation requires the external `codex` CLI on your `PATH`, authenticated
  via `codex login`. See [Codex Engine](/docs/reference/codex-engine).
- `[llm]` is informational under the Codex engine. Codex handles its own
  authentication; `llm.api_key_env` is not used for generation.
- `build.async_runner` controls the pytest marker selected for generated async tests.
- `[context]` and `[context.search]` are covered in depth in [Repo Context](/docs/reference/repo-context).
- `[semantic_gate]` is covered in [Change Detection](/docs/reference/change-detection).
- `[contract]` is covered in the [Contract Mode guide](/docs/guides/contract-mode).
- `[daemon]` is covered in the [Background Daemon guide](/docs/guides/daemon).
- **Changelog (1.2.0):** `[daemon] auto_commit` now defaults to `false` — the
  daemon parks green jobs as reviewable proposals instead of committing them.
  This is the one behavior change existing daemon users feel; set
  `auto_commit = true` to keep auto-commit-on-green.
