---
title: Configuration (jaunt.toml)
description: "Config keys and behavior."
---

Jaunt discovers the project root by walking upward from the current working directory until it finds `jaunt.toml`.

Minimal config:

```toml
version = 1

[paths]
source_roots = ["src"]
test_roots = ["tests"]
generated_dir = "__generated__"
```

Full config (all keys optional except `version`):

```toml
version = 1

[paths]
# Directories to scan for build specs (relative to project root).
source_roots = ["src", "."]
# Directories to scan for test specs (relative to project root).
test_roots = ["tests"]
# Directory name inserted into import paths and used on disk.
# Must be a valid Python identifier. Recommended: "__generated__".
generated_dir = "__generated__"

[llm]
# Informational under the Codex engine. The model that actually runs is set in
# [codex] below, and Codex authenticates via `codex login` / CODEX_API_KEY —
# Jaunt does not read an api_key_env for generation.
provider = "openai"
model = "gpt-5.5"
# Optional hard ceiling for accumulated API spend during a build/test command.
# max_cost_per_build = 5.0

[build]
# Max parallel workers for build generation.
jobs = 8
# Best-effort dependency inference (explicit deps always apply).
infer_deps = true
# Extra ty-based repair passes after the normal retry loop.
ty_retry_attempts = 1
# Pytest marker backend for generated async tests: "asyncio" or "anyio".
async_runner = "asyncio"
# Include targeted test-spec source in build prompts.
include_target_tests = false
# Optional project-wide steering appended to build prompts.
# instructions = ["Prefer small composable helpers."]

[test]
# Max parallel workers for test generation.
jobs = 4
# Best-effort dependency inference for tests.
infer_deps = true
# Passed through to pytest as repeated "--pytest-args" flags.
pytest_args = ["-q"]

[agent]
# Code-generation engine. "codex" is the only supported value.
engine = "codex"

[codex]
# Model passed to `codex exec --model`.
model = "gpt-5.5"
# Reasoning effort: "low", "medium", or "high".
reasoning_effort = "high"
# Codex sandbox policy (`codex exec --sandbox`).
sandbox = "workspace-write"
# Optional Codex feature flags.
# features = []
# Raw `-c key=value` passthrough to `codex exec` (advanced).
# [codex.config]

[prompts]
# If set, these are treated as file paths and read at runtime by the engine.
# Leave empty to use packaged defaults under src/jaunt/prompts/.
build_system = ""
build_module = ""
test_system = ""
test_module = ""
```

Notes:

- `paths.source_roots`: the CLI picks the *first existing* source root as the output base for generated build modules.
- `prompts.*` are treated as *file paths* and read at runtime by the engine.
- Jaunt is batteries-included: `rich`, `watchfiles`, `pytest`, `pytest-asyncio`, and `anyio` ship as core dependencies. There is nothing to install for watch mode, tests, or rich output.
- The only optional extra is `jaunt[mcp]` (for `jaunt mcp serve`).
- Generation requires the external `codex` CLI on your `PATH`, authenticated
  via `codex login`. See [Codex Engine](/docs/reference/codex-engine).
- `[llm]` is informational under the Codex engine. Codex handles its own
  authentication; `llm.api_key_env` is not used for generation.
- There is no `[mcp]` config section; MCP is started explicitly via `jaunt mcp serve`.
- `build.async_runner` controls the pytest marker selected for generated async tests.
