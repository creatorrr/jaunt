This is the shortest path to an "aha": write one tiny spec, run `jaunt build`, and call real generated code like normal Python.

<Callout type="warning">
  `jaunt build` drives the OpenAI **Codex CLI** (`codex exec`) and will spend tokens. Install the `codex` CLI and run `codex login` first.
</Callout>

## Project Layout

Create a small "src layout" project. The `__init__.py` files can be empty:

```text
myproj/
  jaunt.toml
  src/
    my_app/
      __init__.py      # empty
      specs.py          # your @jaunt.magic specs
  tests/
    __init__.py        # empty
    specs_email.py     # your @jaunt.test specs
```

## 1) Install

Jaunt drives the OpenAI **Codex CLI** as its generation engine, so you need
the `codex` CLI installed and authenticated. Install it (see the OpenAI Codex
docs for your platform), then:

```bash
codex login
```

Then install Jaunt from PyPI. The base install is batteries-included
(pytest, pytest-asyncio, anyio, rich, and watchfiles):

```bash
pip install jaunt
# or, in a uv-managed project (one with a pyproject.toml — run `uv init` first
# if you don't have one):
uv add jaunt
```

If you’re working from the Jaunt repo itself:

```bash
uv sync
uv run jaunt --version
```

Codex handles its own authentication (`codex login`, or the `CODEX_API_KEY`
env var) — there is no provider SDK or API key for Jaunt itself to configure.

## 2) Create `jaunt.toml`

`myproj/jaunt.toml`:

```toml
version = 1

[paths]
source_roots = ["src"]
test_roots = ["tests"]
generated_dir = "__generated__"
```

## 3) Write Your First Spec (`@jaunt.magic`)

`myproj/src/my_app/specs.py`:

```python
from __future__ import annotations

import jaunt


@jaunt.magic()
def normalize_email(raw: str) -> str:
    """
    Normalize an email address for stable comparisons.

    Rules:
    - Strip surrounding whitespace.
    - Lowercase the whole string.
    - Must contain exactly one "@".
    - Local-part and domain must both be non-empty after splitting.

    Errors:
    - Raise ValueError if `raw` is invalid by the rules above.
    """
    raise RuntimeError("spec stub (generated at build time)")
```

This docstring is the contract. The type hints matter. The body is a placeholder.

## 4) Generate Code

From `myproj/`:

```bash
jaunt build
```

Jaunt reads your spec, drives `codex exec` to generate the code, validates the output, and writes a real module to:

```text
src/my_app/__generated__/specs.py
```

## 5) Use It (Runtime Forwarding)

You import the spec module. The decorator forwards your call into `__generated__/`.

```bash
PYTHONPATH=src python - <<'PY'
from my_app.specs import normalize_email

print(normalize_email("  A@B.COM  "))  # -> "a@b.com"
PY
```

If you call it before building, you’ll get `jaunt.JauntNotBuiltError` with a hint to run `jaunt build`.

## 6) Add A Test Spec (`@jaunt.test`)

`myproj/tests/specs_email.py`:

```python
from __future__ import annotations

import jaunt


@jaunt.test()
def test_normalize_email__lowercases_and_strips() -> None:
    """
    Assert normalize_email:
    - strips surrounding whitespace
    - lowercases
    - rejects invalid inputs like "no-at-sign" (ValueError)

    Examples:
    - normalize_email("  A@B.COM  ") == "a@b.com"
    """
    raise AssertionError("spec stub (generated at test time)")
```

Now generate tests and run pytest:

```bash
PYTHONPATH=src jaunt test
```

Jaunt writes generated tests under:

- `tests/__generated__/specs_email.py`

## 7) Optional: Tune The Codex Engine

Codex is the only engine, but you can tune the model, reasoning effort, and
sandbox in `jaunt.toml`:

```toml
[codex]
model = "gpt-5.5"
reasoning_effort = "high"   # low | medium | high
sandbox = "workspace-write"
```

The commands stay the same:

```bash
jaunt build
PYTHONPATH=src jaunt test
```

See the [Codex Engine](/docs/reference/codex-engine) reference for details.
